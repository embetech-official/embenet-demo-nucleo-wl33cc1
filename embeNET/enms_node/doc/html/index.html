ENMS: embeNET Node Management Service

This page contains documentation of the embeNET Node Management Service (ENMS) service and its C API library.

Introduction

The ENMS service is a UDP based service running over the embeNET wireless communication protocol. The main function of the service is to provide telemetry, that allows field nodes to report various metrics related to the operation of the network itself and the network services in a unified way.

Architecture

The service consists of the part that runs on the nodes (see ENMS Node API) and the part that runs on the border router or in general outside of the wireless network (see ENMS BorderRouter API). Both services work on top of a UDP port that is registered when the service is initialized.

Functionality

Once started, the ENMS service running in nodes periodically sends out various status indications covering:

  • the identity of the node device hardware and the embeNET stack version
  • detailed network connection related metrics
  • list of network neighbors
  • list of registered network services and their state

The policy that governs how often these indications should be sent can be altered by the application code.

API

This library has two separate APIs:

  • API for the field node : ENMS Node API
  • API for the border router : ENMS BorderRouter API
